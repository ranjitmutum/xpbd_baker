# XPBD 跨模型通用使用说明

本版本不依赖 `Right_Line`、`idle` 或任何固定骨名。模型只要是标准 Bedrock/YSM
geometry + animation JSON，并且骨骼父级关系有效，就可以使用同一套流程。

## 1. 选择求解模式并建立物理骨组

1. 加载模型和动画 JSON。
2. 在右侧“求解模式”选择：
   - `XPBD（兼容）`：默认模式，兼容旧项目和没有 cube 的控制骨。
   - `Bullet 刚体（cube碰撞）`：每根物理骨以自己的有效 cubes 组成一个 compound body，适合需要骨段宽面碰撞、摩擦和反弹的衣摆、头发或配件。
3. 在骨骼树中把飘带、头发、衣摆等连续父子骨加入物理组。
4. 每条链最靠近身体的一根设为“固定（根骨骼）”。
5. XPBD 分叉会自动增加兄弟闭合约束并用多个子方向恢复旋转；刚体模式会在连续父子物理骨的 child pivot 建立 6DoF spring joint，均不依赖特殊骨名。

刚体模式优先使用每根物理骨自身的非退化 cubes。若物理骨自身没有有效 cube，则会把其未被其他物理骨接管的子组 cubes 合成为该骨的 compound body；遇到另一个已勾选的物理骨时停止下探，避免同一 cube 被重复使用。整棵可接管子树都没有有效 cube 时才会在开始烘焙前明确报出骨名。身体碰撞根中的空控制骨可以跳过，其有 cube 的后代仍会正常生成运动学碰撞体。

默认刚体子步是 2，即外层仍按 60Hz 录帧、Bullet 内部固定 120Hz 求解。质量、最大弯曲偏差、角度约束、重力、风、空气响应、湍流和逐骨倍率会继续生效；刚体专用的关节刚度/阻尼、摩擦、弹性、CCD 与穿透阻断值在求解模式下方设置。

## 2. 环境风与角色移动

- **环境风速/方向**：世界里的自然风。
- **改用 XYZ 风分量**：启用后可直接填写环境风 X/Y/Z，并完全替代“风速+方向+仰角”，不会重复相加。X/Z 控制水平吹向，Y 正值向上、负值向下。
- **角色移动速度/方向**：游戏实体的移动；这通常不在动画 JSON 中，必须按动作手动填写。
- 求解器实际使用：`有效相对风 = 环境风 - 角色移动速度`。
- 例：角色沿 +X 以 4 单位/秒前进、无自然风时，环境风速填 0，角色速度填 4、方向填 0°；有效风会自动成为 -X，飘带向后拖曳。
- 右侧会实时显示有效相对风的速度和 X/Y/Z 分量，可直接检查是否同向填反。

例如只想让飘带向上并向 -Z：启用 XYZ 模式，填写 `X=0、Y=1.2、Z=-4`。如果还填写了角色速度，最终仍按 `XYZ环境风 - 角色速度` 计算。

不同模型尺度较大时，先按“模型身高 / 32”同比例缩放风速和角色速度，再微调空气响应。

## 3. 与 idle/stand 动画衔接

1. 在“衔接参考”下拉框中选择这个模型自己的 idle、stand 或其他目标动画。
2. “衔接秒数”建议先用 0.20～0.35 秒；0 表示关闭。
3. 程序先在当前动画中模拟到切出点，再在同一个物理会话中切换到目标动画；导出内容是独立的 A→B 衔接片段。
4. 切换时粒子/刚体、线速度、角速度和现有接触不会重新初始化；目标偏差按临界阻尼衰减，旋转走四元数最短路径。
5. 物理骨 position 与 rotation 都会写入导出 JSON，因此预览与导出结果一致。

动画名称不限；下拉框来自当前加载的 animation JSON。如果参考动画在另一个文件中，先把它合并到同一个 animation JSON 再选择。

## 4. 通用调参起点

- 柔软度：`1e-6`
- 阻尼：`1e-5`
- 迭代次数：8～12
- 动画跟随：0.08～0.2（数值越小，拉回越强）
- 空气响应：0.8～2.0
- 湍流：0.3～1.2

先把环境风设为 0，只调角色移动速度确认拖曳方向；再叠加环境风。这样最容易区分“动作产生的逆风”和自然风。

## 5. 循环判定与首尾连接

- **自动（读取JSON）**：使用当前动画的 `loop` 字段；`hold_on_last_frame` 会按单次模拟并在导出中保留。
- **强制循环**：适合资源漏写 `loop:true` 的情况；模拟会自适应运行多个完整周期，导出写入 `loop:true`。
- **强制单次**：忽略 JSON 的循环标记，只烘焙一次并导出 `loop:false`。

右侧“循环判定”会显示收敛轮数、位置/旋转/速度误差和误差最大的骨骼。每周期使用严格恒定的物理步长；达到阈值后导出最后一个完整物理周期，达到上限仍未收敛时会明确显示并按配置使用缝合回退。

## 6. 碰撞

两种求解模式共用同一套“碰撞根”选择，但碰撞含义不同。

### XPBD 兼容模式

XPBD 提供极简“物理骨 pivot 顶点—身体 cube 六平面”防穿模：

1. 在骨树中单击身体、腿或衣物等障碍骨名称，不要勾成物理骨。
2. 在右侧“身体碰撞”区点击“加入碰撞根”；该骨及其后代的有效 cube 会在初始化时展开。
3. 设置“节点厚度”（默认 `0.1`），重置或播放重新烘焙。

碰撞根为空时碰撞完全关闭。物理骨及位于物理骨子树下的 cube 会从障碍物中排除，避免自撞。求解器把每个物理骨 pivot 当作带 `skin` 的点，对 cube 的六个外向平面做硬推出，并用轻量相对扫掠拦截单步外→外穿盒；推出后只去除相对身体的向内法向速度，因此没有反弹或隐式摩擦。

这不是旧胶囊/OBB/接触流形系统：不保护骨段宽面，不做自碰撞、地面碰撞、三角网格、摩擦或严格旋转 CCD。碰撞骨及祖先的非恒等动画 scale 会被明确拒绝；cube 的正负 `inflate` 会同时作用于预览和碰撞，收缩成零厚度的退化 cube 会跳过碰撞，收缩成负有效尺寸则视为非法输入。最终 loop/衔接后若重新产生穿透，程序会阻止导出。

### Bullet 刚体模式

- 每根物理骨自己的 cubes 合成一个 dynamic/kinematic compound-box body；固定骨跟随原动画，其他骨由 Bullet 独占求解。
- 所选碰撞根展开后的有效 cubes 生成动画驱动的 kinematic bodies；物理骨及其子树仍会从身体碰撞集合排除，避免同一骨同时被两个所有者驱动。
- 支持 compound 宽面接触、摩擦、弹性、动态体 CCD、运动学 child-box sweep、接触流形和父子 spring joint。相邻关节体会禁用互撞，非相邻体仍可接触；当前不生成三角网格或地面，也未提供可配置的整链自碰撞过滤。
- 非恒等动画 scale 不受支持；超过“穿透阻断值”会阻止导出，并在右侧诊断中显示 Bullet 版本、body/cube 数、扫掠、当前接触和最大穿透。

两种模式最终都会生成相同的 `BakedFrame`。求解保持固定步长，完成后仍按原动画关键帧推断公共 FPS（20/24/25/30/40/48/50/60/72/90/120/144/240）并吸附输出时间，非物理骨的原关键帧由导出器保留。

## 7. 约束稳定性改动

- 普通父子段使用 `DistanceConstraint`，阻尼读取每次迭代的隐式速度 `(position-prevPosition)/dt`，不会重复施加同一份外部缓存速度。
- 静止长度接近 0 的重合 pivot 改用逐 XYZ 轴的 `WeldConstraint`，不需要距离归一化和不稳定法线。
- 动画跟随使用求解器内部 `TargetConstraint`，与结构约束共同迭代，不在步进前强改位置。
- 旧角度反三角约束改为 A–C 跨距范围弹簧，避开 0°/180° 共线奇点。
- 固定骨保留由动画写入的位置历史和速度，能向相邻动态骨传递运动学动量。
