# XPBD Bone Baker 中文使用指南

本指南对应当前 JavaFX 桌面版本。软件用于给 Bedrock/YSM 模型的骨骼链加入二级运动，支持默认 XPBD 兼容模式和 Bullet 刚体 cube 碰撞模式；预览后可导出烘焙动画 JSON，也可以额外导出线速度缓存。

> Bullet 刚体模式已接入界面、预览、正式烘焙与导出，使用每骨骼 compound box、
> dynamic CCD、运动学目标 sweep 诊断和父子 6DoF spring joint。XPBD 仍为默认模式，
> 用于兼容旧模型和轻量骨骼链。运动学 sweep 只记录高速穿越风险，不会擅自截断动画目标。

当前版本的循环收敛会同时检查位姿、速度、接触集合、穿透与数值异常；A→B 衔接会保留同一求解器会话中的物理状态，并连续匹配动画驱动的线速度和角速度。最终导出前还会针对处理后的每一帧重新执行碰撞安全审计。

## 1. 运行要求

- Windows x64。
- 64 位 Java 17 或更新版本。
- 必须完整解压发行 ZIP，保留 `run.bat`、`find-java.ps1`、主程序 JAR 和 `lib` 文件夹。
- 双击 `run.bat` 启动；启动器会自动搜索可用 Java，通常不需要设置 `JAVA_HOME`。

## 2. 最短使用流程

1. 选择 `File -> Open Model (.geo.json)`，加载模型 geometry JSON。
2. 选择 `File -> Open Animation (.animation.json)`，加载动画 JSON。
3. 在左下动画列表选择要烘焙的动画。
4. 在左上骨骼树勾选一条连续的飘带、头发或衣摆骨链。
5. 让链条最靠近身体的一根作为固定锚点，其余骨骼保持动态。
6. 在右侧选择求解模式，再设置风、重力、约束和动画跟随参数。
7. 点击底部重置按钮重新初始化，再点击播放；完整烘焙和最终审计在后台运行。
8. 进度到达末帧且安全审计通过后，选择导出烘焙动画；需要速度时再导出速度缓存。

建议先用一条短链测试，确认骨骼方向和固定点正确，再处理完整模型。

## 3. 界面说明

### 3.1 左上：骨骼树

- 单击骨骼名称：只是在右侧选择当前骨骼，用于查看单骨参数或设为碰撞根。
- 勾选骨骼前的复选框：把该骨骼加入物理组。
- 父子复选框相互独立，勾选父骨不会自动勾选全部子骨。
- 物理链应尽量连续。中间漏掉一根会产生两条独立链，并把缺口之后的第一根自动视为固定锚点。

### 3.2 左下：动画列表

- 显示当前 animation JSON 中的所有动画。
- 单击动画名称后，它会成为预览、模拟和导出的源动画。
- 更换动画会使旧烘焙失效，需要重新播放或重置。

### 3.3 中间：三维视图

- 鼠标左键拖动：旋转视角。
- 鼠标右键或中键拖动：平移视角。
- 鼠标滚轮：缩放。
- 蓝色物理点：动态物理骨。
- 红色物理点：固定锚点。

### 3.4 底部控制栏

- `▶|`：前进一步，便于观察单帧变化。
- `▶`：播放模拟；没有物理骨时只预览源动画。
- `⏸`：暂停预览；后台烘焙或文件任务运行时会变为 `Cancel`，用于安全取消。
- `↺`：按当前参数重新初始化模拟。
- 进度条与帧数：显示烘焙进度。
- `Export Animation`：导出烘焙动画；必须先完成整段模拟。

后台任务运行时，模型、动画、参数、播放、步进、重置和导出都会暂时锁定。取消或失败会清除未完成结果，避免把旧帧误当作当前烘焙。

## 4. 物理骨和固定锚点

每条独立物理链必须有一个锚点跟随源动画，否则整条链会自由掉落。当前程序采用以下规则：

- 勾选“固定（根骨骼）”时，当前骨明确固定。
- 当前骨的父骨没有加入物理组时，当前骨会被自动固定，即使界面上的“固定”没有勾选。
- 因此只勾一根骨时，这根骨一定是锚点，不会响应风、重力、质量、湍流或动画跟随。
- 若希望某根可见骨动态运动，应把它的父骨也加入物理组，并让父骨作为锚点。

例如要让 `Ribbon1 -> Ribbon2 -> Ribbon3` 摆动，应至少把三根都加入物理组，让 `Ribbon1` 固定，`Ribbon2/3` 动态。

## 5. 全局物理参数

未启用单骨覆盖的物理骨都使用全局参数。

### 5.0 求解模式

- `XPBD（兼容）`：默认模式，以物理骨 pivot 处理距离、弯曲和选中身体 cube 的碰撞，适合原有项目与轻量骨链。
- `Bullet 刚体（cube碰撞）`：把每根物理骨自己的有效 cube 编译为 compound body，支持宽面接触、摩擦、弹性、CCD、运动学 sweep 诊断和父子关节。若物理骨自身没有有效 cube，程序会自动收集其未被其他物理骨接管的子组 cubes；遇到另一个已勾选的物理骨时停止下探，避免重复碰撞体。
- Bullet 子步默认为每个 60Hz 输出步 2 次，即固定 120Hz。快速运动或接触不稳定时可增加子步，但耗时会提高。
- 模型单位比例默认为 `1/16`。除非模型使用不同尺寸约定，不建议随意修改。
- 切换模式会使现有结果失效，需重新播放完成烘焙。

选择 Bullet 模式后，以下专用控件才会启用：

| 参数 | 默认值 | 含义与调节方向 |
|---|---:|---|
| 刚体子步 | `2` | 每个输出步内部的 Bullet 固定子步，范围 `1～16`。高速、细长 cube 或接触不稳定时可提高。 |
| 刚体单位比例 | `0.0625` | Bedrock 模型单位到 Bullet 世界单位的比例，即默认 `1/16`。 |
| 关节刚度 | `12` | 父子 6DoF spring joint 的角向恢复强度。只在“启用角度约束”时生效。 |
| 关节阻尼 | `0.8` | 父子刚体关节的角向阻尼；过低易振荡，过高会显得僵硬。 |
| 刚体摩擦 | `0.5` | Bullet 接触面的切向摩擦。 |
| 刚体弹性 | `0` | Bullet 法向恢复系数，范围 `0～1`；值越大越容易反弹。 |
| 穿透阻断值 | `0.2` | Bullet 世界单位。模拟或最终帧的最大穿透超过该值时禁止导出。 |
| 刚体启用 CCD | 开启 | 为动态刚体启用连续碰撞检测，减少细长 cube 高速穿透；不能替代合理子步。 |

### 5.1 质量与距离约束

质量在 XPBD 和 Bullet 两种模式中都有效；“柔软度”和“阻尼”是 XPBD 距离约束参数，Bullet 模式改用上方的关节刚度/阻尼。

| 参数 | 默认值 | 含义与调节方向 |
|---|---:|---|
| 质量 | `1.0` | 动态粒子/刚体质量。质量差会影响链条响应；固定骨不使用动态质量。 |
| 柔软度 | `0.000001` | XPBD 距离约束柔度。`0` 最硬；数值越大越容易拉长。长链建议从 `1e-7～1e-5` 调节，过大会散架。 |
| 阻尼 | `0.00001` | 约束方向的速度耗散。增大可减少来回弹动，但过大会显得迟钝。建议从 `1e-5` 附近逐步调节。 |

柔软度控制骨点之间的距离是否容易变化，不等同于“弯曲是否柔软”。想让链条容易弯、但不拉长，应保持较小距离柔软度，再调角度参数。

### 5.2 角度约束

XPBD 用三骨链的跨度约束限制弯曲；Bullet 用相同的“最大弯曲偏差”作为父子刚体关节的对称角度范围，并使用专用关节刚度/阻尼。“角度柔软度”只作用于 XPBD。

| 选项/参数 | 默认值 | 含义与调节方向 |
|---|---:|---|
| 启用角度约束 | 开启 | XPBD 为连续三骨链建立弯曲限制；Bullet 为相邻父子刚体启用角度范围和弹簧刚度。 |
| 最大弯曲偏差° | `75` | 相对模型静止夹角允许偏离的最大角度。越小越接近原姿态，越大越能自由弯曲。 |
| 角度柔软度 | `0.00001` | 角度约束柔度。`0` 最硬；数值越大，达到弯曲上限后的限制越柔和。 |

角度覆盖属于三骨链中间的“关节骨”。把弯曲参数设在没有物理子骨的叶骨上通常没有效果。

### 5.3 碰撞（XPBD 节点参数）

| 选项/参数 | 默认值 | 含义与调节方向 |
|---|---:|---|
| 节点厚度 | `0.10` | 把每个物理骨 pivot 视为带此安全厚度的点。越大越不易让节点进入身体 cube，但也越容易悬空或被多个 cube 夹住。 |
| XPBD 碰撞弹力 | `0` | XPBD 碰撞后的法向恢复系数，范围 `0～1`。`0` 不反弹，值越大越容易弹开。Bullet 模式使用“刚体弹性”。 |

碰撞根非空时启用，清空全部碰撞根时完全关闭。模型 cube 自带的正负 `inflate` 会按 Bedrock 语义同时作用于预览和碰撞；“节点厚度”是 XPBD 物理 pivot 的额外安全距离，建议从 `0.05～0.20` 开始。Bullet 模式直接使用编译后的 cube box 尺寸和材质参数，不使用节点厚度扩张刚体。

### 5.4 重力

| 参数 | 默认值 | 含义 |
|---|---:|---|
| 重力 | `-9.8` | 世界 Y 轴重力加速度。负值向下，`0` 无重力，正值向上。 |

单骨“重力倍率”会乘在此值上。例如全局 `-9.8`、单骨倍率 `0.5`，该骨使用一半重力；全局重力为 `0` 时，任何倍率仍为零。

### 5.5 环境风方向模式

| 参数 | 默认值 | 含义 |
|---|---:|---|
| 环境风速 | `6.0` | 世界空气速度大小。 |
| 水平风向° | `20` | 水平风向：`0°=+X`、`90°=+Z`、`180°=-X`、`-90°=-Z`。 |
| 风向仰角° | `20` | `0°` 水平；正值向上，负值向下；范围 `-90°～90°`。 |

### 5.6 XYZ 风分量模式

勾选“改用 XYZ 风分量”后，环境风 X/Y/Z 会完全替代风速、水平风向和仰角，不会与上方风向模式相加。

| 参数 | 默认值 | 含义 |
|---|---:|---|
| 环境风 X | `0` | 正值吹向 `+X`，负值吹向 `-X`。 |
| 环境风 Y | `0` | 正值向上，负值向下。 |
| 环境风 Z | `0` | 正值吹向 `+Z`，负值吹向 `-Z`。 |

例如想让飘带同时向上和 `-Z`：启用 XYZ，设置 `X=0、Y=1.2、Z=-4`。

### 5.7 角色移动与有效相对风

| 参数 | 默认值 | 含义 |
|---|---:|---|
| 角色移动速度 | `0` | 游戏实体在世界中的移动速度。普通动画 JSON 通常不包含这项，需要手动填写。 |
| 移动方向° | `0` | `0°=+X`、`90°=+Z`，规则与水平风向相同。 |
| 移动仰角° | `0` | 正值表示角色向上移动，负值向下。 |
| 有效相对风 | 自动显示 | 实际求解风为：`环境风 - 角色移动速度向量`。标签同时显示速度大小和 X/Y/Z。 |

让飘带向角色运动后方拖曳的常用设置：环境风设为 0，把“角色移动速度/方向”填写成角色实际运动。程序会自动产生反方向相对气流，不需要把自然风方向手动反填。

### 5.8 空气响应与湍流

| 参数 | 默认值 | 含义与调节方向 |
|---|---:|---|
| 空气响应 | `2.0` | 物理骨速度追随有效相对风的速度。`0` 关闭稳定相对风响应；建议 `0.5～3`。它不会自动关闭下方湍流。 |
| 湍流 | `1.5` | 可重复的阵风加速度。越大，邻近骨点的抖动和不规则摆动越明显；常用 `0.3～2`。 |

单骨“风力倍率”改变该骨对稳定相对风的响应；全局空气响应为 0 或有效相对风为 0 时，倍率再大也不会产生稳定风效果。单骨“湍流倍率”同理依赖全局湍流不为 0。

### 5.9 求解、动画跟随和循环

| 选项/参数 | 默认值 | 含义与调节方向 |
|---|---:|---|
| 迭代次数 | `8` | XPBD 每帧约束求解次数。越大越稳定、越不容易拉长，但计算更慢。Bullet 使用刚体子步和内部约束求解，不读取此值。 |
| 动画跟随 | `0.1` | 两种模式都会把动态骨拉向动画参考。较小的正值拉回更强，较大更自由；`0` 表示关闭拉回，并非无限强。 |
| 衔接参考 | 当前源动画 | 选择其他动画时生成“当前动画 A → 目标动画 B”的独立物理衔接片段；选择当前源动画时不启用跨动画衔接。 |
| 衔接秒数 | `0.25` | A→B 目标偏差按临界阻尼衰减的时长。`0` 关闭；常用 `0.15～0.4` 秒。 |
| 循环模式 | 自动读取 JSON | 决定是否按循环动画预热、收尾和导出。 |

循环模式含义：

- 自动（读取JSON）：使用源动画的 `loop` 字段；`hold_on_last_frame` 按单次模拟并在导出中保留为“保持末帧”。
- 强制循环：即使 JSON 没写循环，也按循环处理并导出 `loop:true`。
- 强制单次：忽略 JSON 的循环标记，只烘焙一次并导出 `loop:false`。
- “循环判定”标签显示最终使用的结果和原因。

循环模拟采用自适应周期固定点迭代：每周期使用 `动画长度 / ceil(动画长度 / 请求步长)` 的严格恒定步长。默认至少预热 2 周期、最多 12 周期，并要求连续 2 次边界比较通过。比较内容包括位置、旋转、线速度、角速度、接触集合、最大穿透和数值异常；接触对象改变、穿透超过当前模式的安全阈值或出现异常时都不会宣告收敛。达到最大轮数仍未收敛时，界面会显示误差与最大误差骨骼，并明确标记是使用缝合回退还是保留最后一个物理周期。

跨动画衔接与循环是两条独立流程。A 会在同一求解器会话中运行到切出点，再把驱动采样器切换到 B；XPBD 粒子、Bullet 刚体、速度与接触世界不会重新初始化。位置偏差使用解析临界阻尼衰减，旋转使用四元数最短路径，并在切换点匹配 A/B 动画驱动的局部线速度和角速度。默认完整权重、零初始偏差速度时，指定时长末端约剩 1% 偏差。衔接参考动画必须存在于当前加载的 animation JSON 中。

## 6. 单骨骼覆盖

### 6.1 全局参数与单骨参数的优先级

- 质量、柔软度、阻尼、弯曲偏差、角度柔软度、动画跟随：勾选覆盖后，单骨绝对值优先；未勾选时使用全局值。Bullet 会读取质量、弯曲偏差和动画跟随，但不会读取 XPBD 专用的距离柔软度、距离阻尼和角度柔软度。
- 重力倍率、风力倍率、湍流倍率：它们不是新的全局值，而是对对应全局效果的单骨倍率。
- 全局参数不会在播放时清除单骨覆盖。
- 单骨覆盖只保存在当前运行内存中；重新加载模型或退出程序后会清空。

### 6.2 应用方法

1. 在左侧单击要设置的骨骼名称。
2. 确保该骨已经勾选进入物理组。
3. 勾选参数名称左侧的覆盖复选框。
4. 输入数值。
5. 点击“应用到骨骼”。只改数值但不点击此按钮，不会保存覆盖。
6. 点击重置或播放重新烘焙。

### 6.3 每个单骨选项

| 选项 | 含义 |
|---|---|
| 固定（根骨骼） | 明确把当前骨设为动画锚点。取消勾选表示恢复“自动判断”，目前不能强制让独立链根变成动态骨。 |
| 质量 | 覆盖该粒子的质量。固定骨不受质量影响。 |
| 柔软度 | 覆盖 XPBD“父骨到当前骨”这条传入距离约束；Bullet 不读取。链根没有父级距离约束，因此设置链根通常无效果。 |
| 阻尼 | 覆盖同一条 XPBD 传入距离约束的速度阻尼；Bullet 不读取。 |
| 弯曲偏差° | XPBD 中当前骨作为三骨链中间关节时生效；Bullet 中作为子刚体的父子关节角度范围生效。 |
| 角度柔软度 | 当前骨作为中间关节时，覆盖 XPBD 角度限制柔度；Bullet 使用全局关节刚度/阻尼。 |
| 动画跟随 | 覆盖当前动态粒子的动画拉回柔度；`0` 关闭拉回。固定骨始终直接跟随动画。 |
| 重力倍率 | `0` 关闭该骨重力，`1` 使用全局重力，`2` 使用两倍重力。 |
| 风力倍率 | `0` 不响应稳定相对风，`1` 正常响应，大于 `1` 更快追随风速。 |
| 湍流倍率 | `0` 关闭该骨阵风，`1` 使用全局湍流，大于 `1` 增强阵风。 |

如果一个参数勾选后仍看不出差异，先检查当前骨是否是红色固定点、是否有对应的传入/角度约束，以及对应全局风/重力/湍流是否为零。

## 7. 碰撞根

物理组和碰撞根是两套独立集合：物理组决定“谁运动”，碰撞根决定“哪一整棵骨骼子树作为障碍物”。

### 7.1 碰撞根按钮

| 按钮 | 含义 |
|---|---|
| 加入碰撞根 | 把当前骨作为根；它及全部后代 cube 自动参与碰撞。 |
| 移出碰撞根 | 移出当前根及其自动后代覆盖。 |
| 清空碰撞根 | 删除所有模型障碍物根选择。 |

碰撞根非空即启用；不再有单独开关。碰撞根子树中的物理骨及其后代会从身体障碍物中排除，避免模拟子树与自身参考 cube 相撞。

### 7.2 推荐流程

1. 在骨骼树勾选飘带链作为物理组。
2. 单击身体、腿或衣物骨名称，但不要勾成物理骨。
3. 点击“加入碰撞根”；例如加入 `RightLeg` 会自动覆盖它的腿和脚后代 cube。
4. XPBD 从较小节点厚度开始；Bullet 先保留默认单位比例、摩擦、弹性和 CCD。
5. 如果 XPBD 被推出、夹住或悬空，减少碰撞骨数量或节点厚度；如果 Bullet 穿透或抖动，先增加子步并检查 compound cubes 与穿透阻断值。

XPBD 模式把每个物理骨 pivot 当作带安全厚度的点，把选定身体 cube 转成 8 个顶点和 6 个外向平面。普通内嵌会选择最近出口；重叠 cube 会按联合障碍物寻找可离开全部相关 cube 的候选。每个求解器迭代中碰撞都排在动画目标之后，确保最后一步把节点推向身体外。

XPBD 高速外→外穿盒使用前后身体姿态下的六半空间线段裁剪，属于轻量相对扫掠；对平移可靠，对快速旋转是前后局部坐标线性近似，不宣称严格旋转 CCD。位置投影后移除相对身体的向内法向速度，并按“XPBD 碰撞弹力”决定是否反弹；当前没有切向摩擦。

Bullet 模式把物理骨 cube 编译为真实 compound boxes，把碰撞根 cube 编译为运动学障碍物，并由 Bullet 处理宽面接触、摩擦、弹性和动态 CCD。运动学 sweep 用于记录高速穿越诊断，实际目标仍按动画写入。无自有 cube 的物理骨会刚性接管普通后代的 cubes，直到遇到另一根物理骨。最终安全审计复用这些实际编译形状及其局部变换，避免后代形状漏检。

## 8. 散架阈值与参数计算器

选择一根动态物理骨后，右侧底部显示估算结果：

- 柔软度上限：以父子静止长度最多伸长约 2% 为目标计算。
- 当前柔软度与安全倍率：倍率大于 1 通常表示静态载荷下有余量。
- 相对风速、重力绝对值、湍流和全链质量倍率上限：每次只改变其中一个，其他参数保持当前值。
- 下游载荷：包括当前骨及物理子链的质量、重力、风和湍流估算。

这是静态、保守的调参参考，不包含动画突然加速、碰撞瞬时冲击和复杂分叉的所有峰值。红色表示建议先降低载荷或柔软度，绿色表示当前估算在范围内。

固定锚点没有父级距离约束，因此不会显示普通伸长阈值。

## 9. 导出

### 9.1 Export Baked Animation

- 导出标准 Bedrock/YSM 动画 JSON。
- 物理骨的 position 和 rotation 会写入每帧关键帧。
- 非物理骨仍来自源动画。
- 必须让模拟运行到结尾；未完成烘焙时程序会拒绝导出。
- 循环/过渡处理之后会执行最终碰撞审计；若 XPBD 样本仍位于身体 cube 内，或 Bullet cube 穿透超过配置的安全阈值，程序会阻止导出。
- 时间键通常保留至少四位小数；当输出步长更小时会自动增加精度，避免多个物理帧静默覆盖同一个 JSON 时间点。
- A→B 独立衔接片使用 B 的时间线作为非物理通道参考，导出 `animation_length` 等于衔接片实际长度。

### 9.2 Export Velocity Cache

- 导出独立 `*.velocity.json` sidecar。
- `frame_rate`：最终烘焙输出采样率。求解器保持 60 Hz；若源动画已有足够的关键帧时间信息，输出会自动吸附到源动画时间格（例如白水绘 `idle` 为 24 FPS），速度缓存会写入相同帧率。无法可靠识别时保持 60 Hz。
- `space`：`model`，表示模型空间。
- `units`：`model_units_per_second`。
- 每帧保存各物理骨的 `[x,y,z]` 线速度。
- YSM 不会自动读取此自定义速度文件，需要消费端显式接入。

## 10. 常用参数起点

### 稳定飘带

- 柔软度：`1e-6`
- 阻尼：`1e-5`
- 最大弯曲偏差：`60～90°`
- 角度柔软度：`1e-5`
- 迭代次数：`8～12`
- 动画跟随：`0.08～0.2`
- 空气响应：`1～2`
- 湍流：`0.3～1.2`

### 角色奔跑时向后飘

- 环境风速：`0`
- 角色移动速度：填写游戏中的实际速度
- 移动方向：填写角色前进方向
- 空气响应：`1～3`
- 先关闭湍流确认方向，再逐步加入湍流

### 只想让末端更活跃

- 根骨保持固定。
- 中段使用正常风力倍率 `1`。
- 末端风力倍率可提高到 `1.5～2`。
- 末端湍流倍率可提高，但叶骨弯曲参数通常无效，应调它的父关节。

## 11. 常见问题排查

### 单骨参数没有效果

依次检查：

1. 当前骨是否真的勾入物理组。
2. 当前骨是否因为父骨不在物理组而自动固定。
3. 参数名称左侧的覆盖复选框是否勾选。
4. 是否点击“应用到骨骼”。
5. 是否重置或播放生成了新烘焙。
6. 风力倍率对应的有效相对风和空气响应是否为零。
7. 重力/湍流倍率对应的全局值是否为零。
8. 柔软度/阻尼是否设在链根，或弯曲参数是否设在叶骨。

### 调了风但方向不对

- 查看“有效相对风”的 X/Y/Z，而不是只看水平风向角。
- 角色移动会从环境风中相减。
- XYZ 模式开启时，上方风速/方向/仰角已被禁用并替代。
- 想表现角色运动产生的逆风，填写角色移动方向，不要同时再手动反向一遍。

### 链条散架或拉长

- 降低距离柔软度。
- 提高迭代次数。
- 降低风速、空气响应、湍流或重力。
- 检查物理链是否连续、是否存在错误固定点。
- 检查节点厚度是否过大，或碰撞根子树是否包含了重叠装饰 cube；必要时选择更靠下的根。

### 飘带像被压住或不动

- 检查动态骨是否实际被自动固定。
- 适当增大动画跟随柔度可让物理更自由；若完全不需要拉回可设为 `0`，但要注意当前版本的 `0` 是“关闭拉回”，不是“最硬跟随”。
- 检查多个重叠身体 cube 是否把骨点夹住，并减小节点厚度或缩小碰撞根范围。
- 降低阻尼，确认空气响应不为 0。

### 碰撞没有效果

- 确认碰撞根数量不为 0。
- 确认所选碰撞根自身或后代带有 cube。
- 同时属于物理组的骨不会作为模型碰撞体。
- 修改碰撞根后需要重新烘焙。
- XPBD 模式检查“节点厚度”和“XPBD 碰撞弹力”；Bullet 模式检查物理骨是否有可用 cube/后代 cube、刚体子步、单位比例、CCD、摩擦和弹性。
- Bullet 的 sweep 命中只是诊断，不会把运动学动画目标停在命中点；真正的安全门禁以 Bullet 接触穿透和最终形状审计为准。

## 12. 当前限制

- 有可见 cube 延伸的叶骨会用隐藏端点恢复旋转；没有可用视觉范围的叶骨仍主要继承父级/参考姿态。
- XPBD 模型碰撞不是网格级碰撞；pivot 点与动画 cube 使用轻量相对扫掠，但没有骨段宽面、切向摩擦或严格旋转/三角网格连续碰撞。XPBD 可配置法向反弹。
- Bullet 提供 box 宽面接触、摩擦、弹性和动态 CCD，但仍不是任意凹面或三角网格布料碰撞；运动学 sweep 目前是诊断而不是目标截断器。
- 两种模式都不提供物理链自碰撞或内置地面；只与显式碰撞根子树的 cubes 碰撞。
- 碰撞骨或祖先存在非恒等动画 scale 时会拒绝开始模拟。cube 的正负 `inflate` 已支持；收缩成零厚度的 cube 会跳过碰撞，收缩成负有效尺寸的非法 cube 会在导入时报告错误。
- 数值型 `linear` 与 `catmullrom` 关键帧会按原插值模式读取；非物理骨导出时会保留模式。Molang 表达式和部分 Bedrock 扩展字段仍不能保证完整保留。
- 物理骨选择、碰撞根和单骨覆盖目前没有项目配置保存功能，切换模型或退出后需要重新设置。
- 速度缓存是本项目自定义 sidecar，不属于标准 YSM 动画通道。
